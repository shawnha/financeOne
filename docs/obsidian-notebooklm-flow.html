FinanceOne — Obsidian + NotebookLM 데이터 흐름

거래 데이터가 어떻게 Obsidian 노트가 되고, NotebookLM이 분석하고, CEO 브리핑이 만들어지는지.

1세 도구의 역할

🗃️

FinanceOne (DB + API)

데이터의 원천
  • 거래내역, 분개, 재무제표 저장
  • 매핑 규칙 + 학습 데이터
  • 월 마감 시 데이터 추출 트리거
📓

Obsidian Vault

컨텍스트 저장소 + 연결 허브
  • 월별 재무 요약 노트 (자동 생성)
  • 거래처 노트 (배경, 계약, 패턴)
  • Wikilinks로 모든 것이 연결됨
  • AI 매핑 시 거래처 컨텍스트 제공
🎙️

NotebookLM

심층 분석 + 오디오 브리핑
  • 재무제표 PDF + 노트를 소스로 분석
  • 이상 지출 패턴, 예측 오차 감지
  • CEO용 오디오 브리핑 자동 생성
  • 예측 보정값 제안
핵심 포인트 이 세 도구는 단방향이 아닙니다. FinanceOne → Obsidian → NotebookLM → 다시 FinanceOne으로 돌아오는 자기학습 루프입니다. 시간이 지날수록 AI 매핑 정확도와 예측 정확도가 올라갑니다.

2전체 데이터 흐름

3월 마감 파이프라인 (8단계)

매달 초에 실행. 수동 트리거 또는 cron 자동화.

1
DB에서 데이터 추출

해당 월의 거래내역, 분개, 잔고를 법인별로 조회

scripts/generate_monthly_notes.py --year 2026 --month 3
2
월별 재무 요약 → Obsidian 저장

법인별 현금흐름, 거래 현황, 상위 지출처를 마크다운 노트로 생성. 거래처명은 [[wikilink]]로 자동 연결.

obsidian-vault/월별 리포트/2026/2026-03-한아원코리아.md
3
거래처 노트 자동 생성

이번 달 처음 등장한 거래처가 있으면 노트를 자동 생성. 기존 거래처는 건너뜀.

obsidian-vault/거래처/ANTHROPIC.md
4
CEO 브리핑 노트 생성

3법인 합산 요약 + 이상거래(상위 5%) + 액션 아이템을 하나의 노트로.

scripts/monthly_briefing.py --year 2026 --month 3
obsidian-vault/월별 리포트/2026/2026-03-CEO-브리핑.md
5
재무제표 PDF 생성

FinanceOne API로 재무제표 Excel/PDF Export

GET /api/statements/{id}/export?format=excel
6
NotebookLM에 소스 업로드

notebooklm-py 라이브러리로 3개 소스를 NotebookLM 노트북에 업로드:

1. 이번달 재무제표 PDF
2. Obsidian 최근 3개월 노트
3. mapping_rules 패턴 요약

→ NotebookLM이 소스를 인덱싱 (자동)
7
NotebookLM 분석 + 오디오 생성

NotebookLM에 질의:

"예측 vs 실제 가장 큰 차이 항목은?"
"이상 패턴의 거래처가 있는가?"
"다음달 주의할 고정비 변동은?"

→ CEO용 오디오 브리핑 자동 생성 (팟캐스트 형식, 3분)

8
분석 결과 → Obsidian + DB 반영

NotebookLM 분석 결과를 Obsidian에 저장. 예측 보정값은 forecasts 테이블 업데이트 제안 → 담당자 승인 후 적용.

obsidian-vault/월별 리포트/2026/2026-03-분석.md
→ forecasts 테이블 보정값 제안 (수동 승인)

4AI 매핑 정확도 향상 루프

거래가 들어올 때마다 Obsidian 거래처 노트가 AI 매핑의 정확도를 높입니다.

거래 매핑 과정 (예: "ANTHROPIC SAN FRANCISCO" ₩167,145)
1. mapping_rules 조회
ANTHROPIC% → 51510 SaaS
신뢰도 0.95, 47회 매칭
→
2. Obsidian 노트 조회
거래처/ANTHROPIC.md
"Claude AI 구독, 매달 25일"
→
3. Claude API + 컨텍스트
노트 내용 프롬프트에 추가
→ 신뢰도 0.99
→
4. 확정 후 학습
hit_count++
거래처 노트 업데이트
신뢰도가 낮을 때만 2~3단계 실행 mapping_rules에서 신뢰도 0.9 이상이면 즉시 매핑. 0.9 미만일 때만 Obsidian 노트를 읽고 Claude API를 호출합니다. 대부분의 거래는 1단계에서 끝납니다.

6시간에 따른 효과

1개월
기본 셋업 완료. 월별 노트 자동 생성. 거래처 노트 20~30개 축적. AI 매핑 정확도 ~70%.
3개월
매핑 규칙 안정화. 반복 거래처 95%+ 자동 분류. NotebookLM이 3개월 트렌드 분석 가능. CEO 브리핑에 전월 비교 포함.
6개월
예측 보정 루프 작동. 현금흐름 예측 오차 ±10% 이내. 계절성 패턴 학습 (명절, 분기말). 거래처 노트 100+ 개.
12개월
자기학습 루프 완성. 새 거래처만 수동 확인. 예측 오차 ±5%. 회계법인 자료 제출 자동화. CEO는 오디오만 듣고 의사결정.

7현재 구현 상태

구성 요소 상태 실행 방법
월별 재무 요약 노트 구현 완료 python scripts/generate_monthly_notes.py --year 2026 --month 3
또는 POST /api/notes/generate
거래처 노트 자동 생성 구현 완료 python scripts/generate_monthly_notes.py --year 2026 --counterparties
CEO 그룹 브리핑 구현 완료 python scripts/monthly_briefing.py --year 2026 --month 3
Vault 상태 확인 API 구현 완료 GET /api/notes/status
NotebookLM 소스 업로드 미구현 notebooklm-py 연동 필요. 현재는 수동 업로드.
NotebookLM 분석 질의 미구현 NotebookLM API 또는 수동 질의
오디오 브리핑 생성 미구현 NotebookLM 내 수동 생성 (Audio Overview)
예측 보정값 → DB 반영 미구현 Phase 5에서 n8n 워크플로우로 자동화 예정
AI 매핑 시 Obsidian 컨텍스트 미구현 Claude API 호출 시 거래처 노트 읽기 (Phase 5)
다음 단계 현재 단계 1~4 (데이터 추출 → Obsidian 저장)는 완료. 단계 5~8 (NotebookLM 연동 → 학습 루프)은 notebooklm-py 라이브러리 연동 또는 수동 운영으로 시작 후, Phase 5에서 n8n으로 자동화합니다.
