전체 요약

무엇을 만드는가

한아원코리아 그룹 3개 법인(HOI·한아원코리아·한아원리테일)의 재무를 자체적으로 관리하는 내부 회계 BPO 시스템. 거래내역 업로드 → AI 자동 분류 → 재무제표 생성까지 자동화.

왜 만드는가

회계법인 의존 구조 탈피. 오늘 잔고와 다음달 말 예상 잔고를 30초 안에 파악. 시간이 지날수록 AI 매핑 정확도가 높아지는 자기학습 구조 구현.

어떤 데이터를 다루는가

우리은행·롯데카드·우리카드 거래내역, Mercury(USD), QuickBooks(HOI), ExpenseOne 경비, 수동 예측치. Excel 업로드 기본, API 연동 선택적.

어떤 결과물이 나오는가

현금흐름 대시보드, 재무상태표·손익계산서·합계잔액시산표, 연결재무제표, CEO용 오디오 브리핑, 회계법인 제출용 Excel/PDF.

배경 & 목표

현재 한아원 그룹은 3개 법인의 재무 데이터를 체계적으로 관리하는 시스템이 없다. 회계법인(새로회계법인)에 의존하는 구조는 비용이 높고, 대표가 실시간으로 현금 흐름을 파악하기 어렵다.

핵심 목표 오늘 잔고가 얼마이고, 다음달 말에 얼마가 남을지를 30초 안에 파악할 수 있어야 한다. 동시에 AI가 거래 패턴을 학습해서 매핑과 예측 정확도가 시간이 지날수록 높아지는 구조를 만든다.

법인 구조

HOI Inc. (미국 모회사, Delaware C-Corp)  ← USD

왜 이 도구들인가

핵심 철학 단순한 기록 시스템이 아니라, 사용할수록 더 똑똑해지는 시스템을 만든다. Obsidian + NotebookLM은 AI가 축적된 컨텍스트로 학습해서 매핑과 예측 정확도를 높이는 피드백 루프의 핵심이다.

Obsidian + NotebookLM — 실제 작동 방식

핵심 목적 Obsidian과 NotebookLM은 단순 메모 도구가 아니다. Claude Code가 거래 데이터와 과거 패턴을 연결해서 매핑 정확도와 현금흐름 예측 정확도를 시간이 지날수록 높이는 자기학습 루프의 핵심이다.

QuickBooks API (HOI — 검증 + 초기 AI 학습 데이터)

역할: 검증 + AI 학습 기준점 HOI 회계는 FinanceOne이 직접 처리 (US GAAP). QuickBooks는 두 가지 역할: (1) FinanceOne이 만든 재무제표를 cross-check하는 검증 기준, (2) 초기 AI 학습 데이터 소스 — QuickBooks의 이미 분류된 거래 데이터가 gaap_mapping 및 mapping_rules의 "정답지" 역할. 장기적으로 FinanceOne 정확도가 올라가면 QuickBooks 의존 제거 가능 (Mercury API만으로 전환).

API 유형용도비용

CorePlus API (읽기)재무제표 조회, 거래 조회월 500,000 크레딧 무료 (Builder 티어)

→ 월 1~2회 검증용 조회 수준이라 전부 무료 티어로 충분.
