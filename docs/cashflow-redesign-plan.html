Task 3 — Backend
예상 현금흐름 API (시차 보정 포함)
forecasts + 카드 시차 보정으로 예상 기말 계산. 실제 진행과 비교.
backend/services/cashflow_service.py backend/routers/cashflow.py backend/tests/test_cashflow.py
카드 이중 차감 방지 forecast_expense는 forecasts의 type='out' 전체 합산 (카드 카테고리 포함). 카드 사용액은 별도 항으로 분리 차감. 공식: 기초 + 입금 - 출금 - 카드사용 + 시차보정.
시차 보정 상세
시차 보정 = 전월 카드 사용(확정) - 당월 카드 사용(예상) 예시: 1월 카드 ₩17.7M, 2월 예상 ₩12.3M 보정 = 17.7M - 12.3M = +₩5.4M → 2월에 결제할 카드대금(=1월분 ₩17.7M)이 2월 카드 사용(₩12.3M)보다 ₩5.4M 더 많음 → 즉, 전월 사용 > 당월 사용이면 카드대금 결제가 늘어남 카드별 분리 (드릴다운용): 롯데: 1월 ₩14.7M - 2월 ₩11.5M = +₩3.2M 우리: 1월 ₩3.0M - 2월 ₩0.8M = +₩2.2M
