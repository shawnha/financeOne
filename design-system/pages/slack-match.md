# Slack Match Page Design

## Layout
- 상단: 법인 선택 탭 + 동기화 상태 배너
- 좌측 (6col): Slack 메시지 리스트
- 우측 (6col): 매칭 후보 거래 + 확정 패널

## Header Actions

```
┌─────────────────────────────────────────────────┐
│  Slack 매칭  (H1)         마지막 동기화: 10분 전  │
│  미확정 12건 · 확정 89건    [ Slack 동기화 ] (CTA) │
└─────────────────────────────────────────────────┘
```

- 동기화 버튼: `--accent` (#22C55E), Lucide `RefreshCw` 아이콘
- 마지막 동기화: `--muted-foreground`, 상대 시간 표시
- 건수: 미확정 (`--warning`), 확정 (`--profit`)

## Slack Message Card (좌측)

```
┌─────────────────────────────────┐
│  #경비  · 김민수 · 2025-01-15   │
│  ─────────────────────────────  │
│  스타벅스 아메리카노 4잔          │
│  19,200원                       │
│  ─────────────────────────────  │
│  💬 쓰레드 2개                   │
│    ├ "입금완료 01/16"            │
│    └ "영수증 첨부"               │
│  ─────────────────────────────  │
│  매칭 상태: AI 매칭 (87%)        │
│  [ 확정 ] [ 수동 매칭 ] [ 무시 ] │
└─────────────────────────────────┘
```

- 카드 배경: `--card` (#1E293B)
- 채널명: `--ai-accent` (#6366F1) 배지
- 금액: `font-mono`, 20px, `--foreground`
- 쓰레드: 접힘/펼침, `--muted-foreground`
- 매칭 신뢰도 배지:
  - 90%+: `--profit` (green)
  - 70-89%: `--warning` (yellow)
  - <70%: `--loss` (red)

## Match Candidate Panel (우측)

매칭 후보 거래 1~3건 표시:

```
┌─────────────────────────────────┐
│  매칭 후보                       │
│  ─────────────────────────────  │
│  ◉ 01/15 우리카드 스타벅스        │
│    ₩19,200  신뢰도: 95%         │
│    매칭 근거: 금액 일치 + 날짜    │
│  ─────────────────────────────  │
│  ○ 01/15 롯데카드 스타벅스        │
│    ₩19,200  신뢰도: 72%         │
│    매칭 근거: 금액 일치           │
│  ─────────────────────────────  │
│  [ 선택 확정 ] (btn-primary)     │
│  [ 직접 검색 ] (btn-secondary)   │
└─────────────────────────────────┘
```

- 라디오 버튼 선택 방식
- 신뢰도별 색상 (위 배지 규칙 동일)
- 매칭 근거: `--muted-foreground`, 12px
- AI 추론 로그: 접힘/펼침 (ai_reasoning 필드)

## Match Rules Display

3단계 매칭 규칙 시각화:

| 단계 | 규칙 | 표시 |
|------|------|------|
| 1 | 정확 금액 (±1원) | "금액 일치" 배지 (green) |
| 2 | VAT 역매칭 (×1.1) | "VAT 역산" 배지 (blue) |
| 3 | 수수료 허용 (±3%) | "수수료 범위" 배지 (yellow) |

## Manual Match (직접 검색)

거래내역 검색 모달:
- 날짜 범위 (Slack 메시지 ±7일 기본)
- 금액 범위 (±10% 기본)
- 거래처명 검색
- 결과 테이블에서 클릭하여 매칭

## Filters

- 상태: 전체 / 미확정 / 확정 / 무시
- 기간: 날짜 range picker
- 채널: 드롭다운 (#경비, #일반 등)
- 신뢰도: 높음(90%+) / 중간(70-89%) / 낮음(<70%)

## Interaction States

| State | Implementation |
|-------|----------------|
| LOADING | 메시지 카드 스켈레톤 (3개) + 매칭 패널 스켈레톤 |
| EMPTY | "Slack 동기화를 시작하세요" + 동기화 버튼 + Slack 연결 안내 |
| ERROR | Token 에러 → "Slack 연결이 만료되었습니다. 설정에서 재연결해주세요." + 설정 링크 |
| SUCCESS | 메시지 리스트 + 매칭 결과 정상 표시 |
| PARTIAL | 미확정 매칭 → 노란 배너 "12건의 매칭을 확인해주세요" |

## Sync Status Banner

```
동기화 중...  ━━━━━━━━━━━━━ 67%  (45/67 메시지)
```

- 프로그레스 바: `--accent` (#22C55E)
- 동기화 완료: "67개 메시지 동기화 완료. 12건 새 매칭 발견." (auto-dismiss 5초)
- 동기화 에러: rate limit → "Slack API 제한. 30초 후 자동 재시도합니다."

## Keyboard Shortcuts
- `j/k`: 메시지 리스트 위/아래 이동
- `Enter`: 선택된 매칭 확정
- `m`: 수동 매칭 모달 열기
- `i`: 무시 처리
