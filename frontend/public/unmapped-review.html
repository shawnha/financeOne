미분류 거래 반영 방식 — 설계 리뷰

예상 현금흐름 테이블 + 그래프에서 미매핑 거래를 어떻게 보여줄 것인가

1. 테이블 비교

Option A 정보 표시만 (예상 잔고 미반영)
항목예상실제예상잔고
기초 잔고--161.0M
수입 스마트스토어+226.7M+220.5M387.7M
지출 급여-85.0M-85.0M302.7M
지출 물류비-45.0M-43.8M257.7M
미분류 7건 -- +3.2M / -8.4M 반영 안 됨
기말 잔고--257.7M
미분류 입금 +3.2M, 출금 -8.4M이 예상 잔고에 반영되지 않음.
실제 기말은 252.5M인데 예상은 257.7M으로 표시 → 5.2M 갭이 보이지 않음
Option B 예상 잔고에 반영 (조정된 예상)
항목예상실제예상잔고
기초 잔고--161.0M
수입 스마트스토어+226.7M+220.5M387.7M
지출 급여-85.0M-85.0M302.7M
지출 물류비-45.0M-43.8M257.7M
미분류 7건 -- +3.2M / -8.4M 252.5M
기말 잔고--252.5M
미분류 순효과(-5.2M)가 예상 잔고에 즉시 반영.
예상 기말 252.5M ≈ 실제 기말 252.5M → 갭 원인이 투명하게 보임
월말: "어떤 항목을 예상에서 빠뜨렸나" 바로 파악 가능

2. 일별 시뮬레이션 그래프 비교

Option A 그래프에 미분류 미반영

예상 선과 실제 선의 갭 원인을 알 수 없음

Option B 미분류 반영 → 조정된 예상선 추가

미분류 거래 발생 시점에 조정선이 꺾임 → 갭 원인 투명

3. 월말 분석 시나리오 — Option B가 가능하게 하는 것

Step 1: 월 중간

"미분류 7건 -5.2M"
조정 예상 기말: 252.5M
→ 예상보다 5.2M 적을 것이라는 조기 경보

Step 2: 월말 마감

예상: 257.7M → 실제: 252.5M
차이 -5.2M의 원인:
- 미분류 지출: -8.4M
- 미분류 수입: +3.2M
→ 항목별 차이 분해 가능

Step 3: 다음 달 예상 작성

"이번 달 빠뜨린 항목:"
- 미분류 출금 7건 중 5건 = 복리후생비
→ 다음 달 예상에 복리후생비 항목 추가
→ 예상 정확도 점진적 개선

4. 예상 기말 공식 비교

A현재
예상 기말 = 기초 잔고
  + 예상 입금
  - 예상 출금
  ± 카드 시차보정
B제안
조정 예상 기말 = 기초 잔고
  + 예상 입금
  - 예상 출금
  ± 카드 시차보정
  + 미분류 실제 입금
  - 미분류 실제 출금

결정 필요

Option A: 정보 표시만
미분류 행은 보여주되, 예상 잔고/그래프에는 반영하지 않음. 순수 예상과 실제를 분리.
Option B: 조정된 예상 (추천)
미분류 거래를 예상 잔고에 반영. 그래프에 "원래 예상" 선(흐릿) + "조정 예상" 선(진하게). 월말 분석과 다음 달 예상 정확도 개선 가능.
Option C: 토글 (둘 다)
"미분류 반영" 토글로 사용자가 선택. 기본값 B, 순수 예상 보고 싶으면 A로 전환.
