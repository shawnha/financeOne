현재: Static Worst-Case

forecast 항목만으로 계산. 실제 데이터가 들어와도 변하지 않음.

장점: "원래 계획 기준 최악"을 보여줌
단점: 실제가 worst-case보다 나빠지면 거짓 안심을 줌

제안: Shifted Worst-Case

실제 구간은 실제 잔고, 미래 구간은 마지막 실제 잔고 + 남은 worst-case 변동분.

장점: "지금 시점에서 앞으로 최악은?" — CFO가 필요한 정보
단점: 원래 worst-case 계획과 비교 불가
